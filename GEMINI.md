# Project Workflow Instructions

## Git & Pull Request Policy
For every future code change (feature or bug fix):
1. **Branching:** Create a new descriptive branch from `main` (e.g., `git checkout -b feat/discovery-logic`).
2. **Commit & Push:** Apply changes, commit, and push the branch to GitHub.
3. **Pull Request:** Provide the user with the branch link. The user will manually create and merge the Pull Request after verification.
4. **No Direct Main Commits:** Never push directly to the `main` branch unless explicitly instructed otherwise for an emergency fix.
